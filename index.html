💻 Code Completions — The Classic

Inline suggestions are Copilot's bread and butter. Here's how to use them like a pro:

  • Just start typing — ghost text appears automatically
  • Press Tab to accept the whole suggestion
  • Press → to accept word-by-word (underrated!)
  • Alt+] / Alt+[ to cycle through suggestions
  • Open the completions panel to see up to 10 alternatives
Write comments first! Describe what you want in plain English, then let Copilot write the function below it. It's like pair programming with someone who never complains. 

# Calculate distance between two GPS coordinates
# using the Haversine formula
def haversine(lat1, lon1, lat2, lon2):
    # ✨ Copilot writes the whole thing...
    R = 6371  # Earth radius in km
    dlat = radians(lat2 - lat1)
    dlon = radians(lon2 - lon1)
    a = sin(dlat/2)**2 + cos(radians(lat1)) \
        * cos(radians(lat2)) * sin(dlon/2)**2
    return R * 2 * asin(sqrt(a))

You wrote the comment. Copilot wrote everything else. You're basically a software architect now.

GitHub CLI

For People Who Prefer Typing to Clicking

Terminal warriors, your moment has arrived.

⌨️ GitHub Copilot CLI — The Agentic Terminal

Not just a command suggester — a terminal-native AI coding partner that can work autonomously, delegate tasks, plan implementations, and manage sessions. Invoke it with copilot.

Core Slash Commands

🗺️ /plan (or Shift+Tab)

Generates a structured implementation plan with checkboxes before writing a single line of code. Asks clarifying questions, saves to plan.md, and waits for your approval.

☁️ /delegate

Offloads work to the Copilot cloud agent — creates a PR asynchronously while you keep working locally.

Model Selection with /model

  • 🤖 Auto — intelligently picks the best model, reduces rate limits
  • 🧠 Claude Opus 4.5 (default) — complex architecture, deep debugging
  • ⚡ Claude Sonnet 4.5 — fast, cost-effective, handles most tasks
  • 🔍 GPT-5.2 Codex — code generation, second-opinion reviews
Switch models mid-session as task complexity changes. Start with Auto, switch to Opus for the gnarly bits.
Hackathon tip: Use /help to see all available commands. The CLI is continually evolving and /help always shows the most up-to-date capabilities.

⌨️ The Explore → Plan → Code → Commit Workflow

The golden workflow for complex hackathon features — models achieve higher success rates when given a concrete plan to follow.


# 1. EXPLORE — understand first, don't code yet
Read the auth files but don't write code yet

# 2. PLAN — create a structured plan

# Copilot asks clarifying questions,
# creates a checklist plan, saves to plan.md,

# 3. IMPLEMENT — execute the approved plan
Proceed with the plan

# 4. VERIFY — don't skip this step
Run the tests and fix any failures

# 5. COMMIT — clean history
Commit these changes with a descriptive message

🚀 Prompting That Actually Works

The quality of Copilot's output is proportional to the quality of your prompt. Here's the formula:

❌ Weak Prompts

  • "write a function"
  • "fix this"
  • "make it better"
  • "add auth"

✅ Strong Prompts

  • "Write a TypeScript function that takes an array of user objects and returns them sorted by creation date, with null dates last"
  • "Fix the race condition in this async function — it should wait for all promises before returning"
The CRISPE formula: Context (what is the project?) · Role (act as a senior engineer) · Instructions (what to do) · Style (TypeScript, functional, etc.) · Purpose (this is for a hackathon demo) · Expected output (a working function with error handling)

🤖 Custom Agents — Your AI Teammates

Custom agents are specialized versions of the Copilot agent — defined once, reused forever. They encode your conventions, frameworks, and desired outcomes directly into Copilot.

Agent Profiles

A Markdown file with YAML frontmatter at .github/agents/NAME.md:

name: api-builder
description: Builds REST API endpoints
  following our conventions
---

You are an API specialist for this
project. Always:
- Use FastAPI with async handlers
- Return { data, error } objects
- Add input validation with Pydantic
- Write tests for every endpoint
- Follow our OpenAPI spec in /docs

Where Custom Agents Work

  • 💻 VS Code, JetBrains, Eclipse, Xcode
  • ⌨️ GitHub CLI

Scope Levels

  • .github/agents/ → repo-level (team shared)
  • Org's .github repo → org-wide
