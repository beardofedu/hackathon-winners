💻 Code Completions — The Classic

Inline suggestions are Copilot's bread and butter. Here's how to use them like a pro:

  • Just start typing — ghost text appears automatically
  • Press Tab to accept the whole suggestion
  • Press → to accept word-by-word (underrated!)
  • Alt+] / Alt+[ to cycle through suggestions
  • Open the completions panel to see up to 10 alternatives
Write comments first! Describe what you want in plain English, then let Copilot write the function below it. It's like pair programming with someone who never complains. 

# Calculate distance between two GPS coordinates
# using the Haversine formula
def haversine(lat1, lon1, lat2, lon2):
    # ✨ Copilot writes the whole thing...
    R = 6371  # Earth radius in km
    dlat = radians(lat2 - lat1)
    dlon = radians(lon2 - lon1)
    a = sin(dlat/2)**2 + cos(radians(lat1)) \
        * cos(radians(lat2)) * sin(dlon/2)**2
    return R * 2 * asin(sqrt(a))

You wrote the comment. Copilot wrote everything else. You're basically a software architect now.

GitHub CLI

For People Who Prefer Typing to Clicking

Terminal warriors, your moment has arrived.

⌨️ gh copilot — Your Shell-Based Pair Programmer

The gh copilot extension brings AI assistance to your terminal.

Installation


# Install gh CLI first (if you haven't)
brew install gh       # macOS
winget install gh     # Windows
apt install gh        # Debian/Ubuntu

# Authenticate
gh auth login

# Install Copilot extension
gh extension install github/gh-copilot

The Two Commands That Matter


# Explain a shell command
gh copilot explain "git rebase -i HEAD~5"

# Suggest a command from English
gh copilot suggest "undo my last commit
  but keep the changes staged"

# Set up convenient aliases
gh alias set cop 'copilot suggest'
gh alias set exp 'copilot explain'
Hackathon tip: You WILL run a command you don't fully understand during the hackathon. Use gh copilot explain before you accidentally git push --force to main.

⌨️ gh copilot in Action

Real scenarios you will definitely encounter during a hackathon:


# "How do I find which process is using port 3000?"
gh copilot suggest \
  "find and kill the process on port 3000"

# Copilot suggests:
lsof -ti:3000 | xargs kill -9
        

# "I need to compress this folder for the demo"
gh copilot suggest \
  "create a tar.gz of the src directory \
   excluding node_modules"

# Copilot suggests:
tar -czf src.tar.gz \
  --exclude='src/node_modules' src/
        

# "What even is this command in our Makefile?"
gh copilot explain \
  "docker run --rm -v $(pwd):/app \
   -w /app node:18 npm run build"

# Copilot explains:
# Runs a temporary Docker container
# using Node 18, mounts your current
# directory, and runs npm build inside
# the container. No local Node needed.
Interactive mode: Run gh copilot suggest without arguments for a conversational shell assistant. It asks clarifying questions and refines suggestions.

🚀 Prompting That Actually Works

The quality of Copilot's output is proportional to the quality of your prompt. Here's the formula:

❌ Weak Prompts

  • "write a function"
  • "fix this"
  • "make it better"
  • "add auth"

✅ Strong Prompts

  • "Write a TypeScript function that takes an array of user objects and returns them sorted by creation date, with null dates last"
  • "Fix the race condition in this async function — it should wait for all promises before returning"
The CRISPE formula: Context (what is the project?) · Role (act as a senior engineer) · Instructions (what to do) · Style (TypeScript, functional, etc.) · Purpose (this is for a hackathon demo) · Expected output (a working function with error handling)

🚀 Custom Instructions — Your Secret Weapon

Set up custom instructions so Copilot always knows your project's context before you ask anything.

Repository Instructions

Create a .github/copilot-instructions.md file:


# Project: HackathonApp

## Stack
- React + TypeScript frontend
- FastAPI Python backend
- PostgreSQL database
- Deployed on Railway

## Conventions
- Use async/await, never callbacks
- All API responses use { data, error }
- Prefer functional components
- Write tests for all API endpoints

## Context
This is a hackathon project for
tracking team productivity metrics.

Personal Instructions

Set in GitHub Copilot settings for your personal style:

  • Preferred programming language style
  • Error handling preferences
  • Testing framework of choice
  • Code comment verbosity level
Set these up before the hackathon starts. Copilot will automatically follow your conventions without you having to explain them every time.
